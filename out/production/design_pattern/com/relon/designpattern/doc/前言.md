# 1.什么是设计模式？

个人认为设计模式就是用来解决普适性问题的指导方案，更侧重于思想。属于解决某一问题的一系列套路，是具体问题的抽象解决方案，了解一下设计模式出现的北京可以帮助更好的理解什么是设计模式

>设计模式”这个术语最初并不是出现在软件设计中，而是被用于建筑领域的设计中。

> > 1977 年，美国著名建筑大师、加利福尼亚大学伯克利分校环境结构中心主任克里斯托夫・亚历山大（Christopher Alexander）在他的著作《建筑模式语言：城镇、建筑、构造（A Pattern Language: Towns Building Construction）中描述了一些常见的建筑设计问题，并提出了 253 种关于对城镇、邻里、住宅、花园和房间等进行设计的基本模式。
> >
> > 1979 年他的另一部经典著作《建筑的永恒之道》（The Timeless Way of Building）进一步强化了设计模式的思想，为后来的建筑设计指明了方向。
> >
> > 1987 年，肯特・贝克（Kent Beck）和沃德・坎宁安（Ward Cunningham）首先将克里斯托夫・亚历山大的模式思想应用在 Smalltalk 中的图形用户接口的生成中，但没有引起软件界的关注。
> >
> > 1990 年，软件工程界才开始研讨设计模式的话题，后来召开了多次关于设计模式的研讨会。
> >
> > 1995 年，艾瑞克・伽马（ErichGamma）、理说隆ず６姆（Richard Helm）、拉尔夫・约翰森（Ralph Johnson）、约翰・威利斯迪斯（John Vlissides）等 4 位作者合作出版了《设计模式：可复用面向对象软件的基础》（Design Patterns: Elements of Reusable Object-Oriented Software）一书，在本教程中收录了 23 个设计模式，这是设计模式领域里程碑的事件，导致了软件设计模式的突破。这 4 位作者在软件开发领域里也以他们的“四人组”（Gang of Four，GoF）匿名著称。
>
> 虽然到如今出现的设计模式已经不止23中，但是狭义上来讲，设计模式就是指GoF的23种设计模式

设计模式就像是一个框架，指导了我们如何更好的组织代码，来获得更好的代码灵活性、重用性、可读性等。

举个不太恰当的例子，比如通过代码计算1+2+3、4+5+6两个加法的值，我们可以单独计算，但是这样肯定就限制了只能计算特定值的表达式，相信都不会这么来做，通常设置一个函数 `x+y+z = n`这样的函数无疑更通用（三数相加的计算都适用），且拓展性更强

2. 提高编程能力、思维能力和设计能力

# 3. 设计模式基本要素

**1. 名称**

即设计模式姓甚名谁

**2. 问题(适用场景)**

我们不可能脱离具体的场景来谈应该使用哪种设计模式，毕竟每一种设计模式都有适合的应用场景，所谓术业有专攻

**3. 解决方案**

确定应该使用的模式后，需要有一个具体的实现方式，通常可以理解为各个类、接口、方法等的组合方式

**4. 效果**

一方面是应用后是否解决当前问题的痛点，是否有效提高了可拓展性、可维护性等，另一方面也应权衡应用后的弊端等等
